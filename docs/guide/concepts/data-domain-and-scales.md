# 数据范围、标度与投影

## 先用一句话说明

标度规定哪些数据值可以出现以及它们之间如何比较，投影再把这些值换算成面板中的像素位置。

在线性轴上，数值差相同就占相同距离；分类轴只关心类别顺序；对数轴让相同比例占相同距离。理解这个区别，才能判断“线看起来挤在一起”究竟是数据问题、范围问题，还是应该换标度。

## 为什么重要

自动范围让第一张图几乎零配置，但它不能替你决定业务含义。温度从 18 到 22 度时，自动范围可能突出细小变化；展示从零开始的销量柱时，同一范围会放大差异。跨六个数量级的数据在线性轴上会把小值压在底部，而对数轴不能表达零和负数。每种选择都改变读者如何比较数据。

如果还没有区分整张图、面板和数据层，先回到[组合模型](figure-axes-series.md)；标度属于面板，而不是某一条系列。

标度还参与交互。缩放和平移改变面板当前可见范围，系列仍保留原始数据；标注使用数据坐标时会跟随投影移动。若把屏幕像素误当数据值，窗口尺寸变化后注释就会失去位置意义。

## 工作模型

每个 `Axes` 有 x 与 y 两个标度。系列先报告自身有效数据范围，面板合并所有系列的范围，再交给标度生成适合显示的域。线性标度通常会得到易读刻度；分类标度把字符串类别映射为顺序位置；对数标度只接受正域，并以比例关系分布。

投影接收数据域和面板绘图矩形，提供数据值与像素之间的双向换算。系列绘制、命中测试和交互缩放共享这一投影，因此调用者通常不需要自己计算像素。真正需要控制的是数据域、标度类型和标签，而不是底层坐标公式。

空系列不会提供有效域。如果一个面板中的全部系列都为空，自动范围没有业务信息可推断；此时应明确决定是显示“暂无数据”、跳过面板，还是提供合理的固定范围，而不是让默认域看起来像真实结果。

## 选择与取舍

- **自动线性范围**：趋势、测量值和大多数连续数据的默认选择。先使用它，再根据业务需要固定基线或上限。
- **显式线性范围**：多个图需要可比尺度，或零点/阈值具有明确含义。固定范围可能裁掉异常值，必须配合数据检查。
- **分类标度**：月份名称、产品类别等离散有序标签。系列数据位置按类别索引解释，不应拿类别间距表示数值差。
- **对数标度**：正值跨多个数量级，需要比较倍数而非差值。零和负值不能被对数变换；应在进入绘图前决定过滤、分组还是改用其他表达。

不要仅因为“图更好看”就切换标度。标度是对关系的声明：线性回答相差多少，对数回答相差多少倍，分类轴回答属于哪个槽位。

## 应用这个模型

先对比分类关系。分类轴把数组位置解释为命名槽位；标签顺序改变时，数据数组也必须同步改变：

```cangjie role=contrast
axes.setXScale(CategoryScale(["一月", "二月", "三月", "四月"]))
axes.add(BarSeries([12.0, 18.0, 15.0, 22.0], label: "销量"))
```

再追踪对数路径：先建立正数域，再把域交给标度，最后把同一域设为可见范围。这样“比例关系”和“当前视图”是两个明确步骤：

```cangjie role=trace
let positiveDomain = Bounds(1.0, 1.0e6)
axes.setYScale(LogScale(positiveDomain))
axes.setYLimits(positiveDomain)
println("对数可见域：${axes.effectiveYBounds()}")
```

在设置前检查数据最小值。若零表示真实的“没有发生”，将它悄悄替换为极小正数会歪曲语义；更好的做法可能是把零单独标注，或保留线性轴。

## 常见误解

- **“自动范围总是最客观。”** 它忠实包围数据，却不理解零基线、跨图可比性或业务阈值。
- **“对数轴只是压缩画面。”** 它改变距离含义，从比较差值变成比较比例。
- **“空数据等于零。”** 空表示没有观测，零是一个有效观测；两者不应共享同一处理。
- **“标注位置是像素。”** 面向数据的标注使用数据坐标，随缩放和平移保持语义位置。
