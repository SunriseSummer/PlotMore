# 数据范围与标度

## 先用一句话说明

数据范围说明有哪些值，标度决定这些值怎样映射到坐标轴。

Plot 还区分“自动得到的有效范围”和“用户交互后的视图范围”。这三个概念如果混在一起，就容易把空图误判为渲染问题，或在缩放后以为原始数据已被改变。

## 为什么重要

坐标轴是读者解释位置的尺子。范围过宽会把变化压平，范围过窄会截掉数据；错误地使用对数标度会让零和负值无处可画；把类别编号当作普通连续数值会让柱形不再对齐类别名称。Plot 会从可见系列合并数据范围并自动留出合理显示域，但业务阈值、跨图比较和特殊标度仍需要应用明确设置。

## 工作模型

系列先通过 `dataBounds` 报告自己的有效 x/y 范围，Axes 合并所有可见系列，再交给 Scale 生成有效域与刻度。如果应用调用 `setXLimits` 或 `setYLimits`，显式范围优先；用户缩放和平移则通过 `setView` 形成临时视图；`resetView` 清除临时视图，回到限制或自动范围。空系列不会贡献有效域，空面板会退回标度本身的域。

下面对比连续数值域和类别域。两段代码都控制横轴，但语义完全不同。

```cangjie role=contrast
axes.setXLimits(Bounds(0.0, 24.0))
axes.setYScale(LogScale(Bounds(1.0, 1000000.0)))
axes.add(LineSeries(hours, operations, label: "操作数"))
```

```cangjie role=trace
axes.setXScale(CategoryScale(["东区", "南区", "西区", "北区"]))
axes.showGridX = false
axes.add(BarSeries([18.0, 24.0, 16.0, 21.0], label: "订单"))
```

## 选择与取舍

自动范围适合探索和大多数单图；显式范围适合需要统一比较多个面板、必须包含业务基线或必须展示阈值的报告。不要仅为了让波动“看起来更大”而截断纵轴，尤其是柱形图。对数标度适合数量级跨度大且比例有意义的数据，只接受正域；含零或负值时，应换回线性标度、调整问题定义或清楚说明数据转换。

类别标度将名称映射到等宽带，BarSeries 会据此确定柱宽。它不是把字符串偷偷变成连续测量。日期在当前 API 中通常仍以数值表示，标签格式需由应用的刻度策略处理。固定范围也不是数据清洗：越界值可能不显示，但原始数组仍在系列中。

## 应用这个模型

排查范围时依次记录原始数组最小/最大值、系列是否可见、Axes 是否设置限制、Scale 当前域、Axes 是否存在交互视图。不要同时修改所有层。要让两张图可比较，可给它们相同的 `Bounds`；要恢复自动行为，调用相应 autoscale 或清除显式设置；要恢复用户操作前的显示，调用 `resetView`。

前置对象关系见[Figure、面板与系列](figure-axes-series.md)。实际设置步骤见[配置坐标轴](../how-to/configure-axes.md)，交互带来的临时范围见[交互与视图状态](interaction-and-view-state.md)。系列先报告自身有效数据范围，面板合并所有系列的范围，再交给标度生成适合显示的域。对数标度只接受正域，并以比例关系分布。

## 常见误解

“设置标度就等于设置限制”不准确：标度负责映射规则，限制负责业务显示边界。`setView` 也不是删减数据，它只是临时显示窗口。标注通常不参与自动范围；若阈值线远离数据，应显式设置范围让它可见。另一个误区是用极端异常值支配所有显示：如果异常值真实且重要，应单独说明或分面，而不是悄悄过滤。

## 相关 API

- [`Bounds`](../../api/plot/core/Bounds.md)：表示数值上下界并提供合并、扩展等操作。
- [`Scale`](../../api/plot/core/Scale.md)：线性、对数和类别映射的共同接口。
- [`Axes`](../../api/plot/axes/Axes.md)：合并范围、设置限制和保存视图状态。
