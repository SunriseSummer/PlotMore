# 样式、图例与输出

## 先用一句话说明

数据决定图形表达什么，样式帮助读者区分和聚焦，窗口或导出器决定结果以何种介质交付。

这三件事经常在一段示例里同时出现，却不属于同一层。把它们分开后，你可以在不改变数据和图表选择的情况下切换主题，也可以把同一 `Figure` 同时用于屏幕查看和 PNG 报告。

## 为什么重要

如果颜色承担唯一含义，打印、色觉差异或主题切换会让信息丢失；如果窗口设置混进图形构建函数，批处理导出就不得不启动交互界面；如果图例只是自动生成却没有清楚的系列标签，读者仍然不知道线条代表什么。良好的交付需要数据语义、视觉编码和生命周期各自清晰。

这一区分也让测试更稳定。构建 `Figure` 可以是确定性的纯步骤，窗口交互由人工或集成测试验证，文件导出检查路径和编码。无需为了测试数据组合而模拟鼠标，也无需为了批量生成图片等待用户关闭窗口。

## 工作模型

系列拥有与自身有关的视觉属性，例如颜色、线型、标记、填充和标签。面板拥有坐标语境中的元素，例如图例位置、网格和数据坐标标注。`Figure` 拥有整张作品的主题、标题和面板排列。优先在正确层设置属性，避免一条系列反向控制整张图。

图例来自带标签的系列。标签说明业务身份，颜色和线型帮助快速匹配；两者应同时存在。阈值线、区间带和文字说明使用数据坐标，适合表达 SLO、目标区间或事件时间。它们补充数据语境，不应伪装成新的数据系列。

`PlotWindow` 消费 `Figure` 并提供交互查看；`FigureExport.renderToPng` 自行创建隐藏窗口并写出 PNG。`FigureExport.savePng`/`saveBmp` 则复用调用者已有的 `Renderer` 和 `Canvas`，不会替调用者创建隐藏窗口。这些路径都读取同一图形描述，因此标题、系列和主题保持一致；窗口附加工具栏或悬停读数不等于导出图本身的内容。

## 选择与取舍

- **系列标签 + 图例**：两条以上需要识别的系列。单条系列若标题已明确，也可减少冗余图例。
- **颜色 + 线型/标记**：需要兼顾色觉、黑白打印或密集重叠；不要只靠细微色差。
- **参考线/区间**：阈值和目标区间属于解释上下文，不是观测数据。
- **交互窗口**：探索、缩放、查看悬停值和人工检查。
- **直接导出**：报告、批处理、自动化产物和无需人工关闭窗口的流程。

主题提供一组协调默认值，而不是替代语义设计。即使使用内建深色主题，仍要检查文本对比、系列数量和图例位置是否适合当前数据。

## 应用这个模型

先对比数据层与解释层：折线携带观测和身份，参考线解释目标，两者进入同一面板却承担不同职责。

```cangjie role=contrast
let latency = LineSeries(minutes, values, label: "p99 延迟")
axes.add(latency)
let slo = HorizontalLine(150.0)
slo.label = "SLO 150 ms"
let _ = axes.annotate(slo)
```

同一 `figure` 可以先供人工检查，也可以交给导出器。下面的调用轨迹刻意复用同一对象：关闭窗口后 `show()` 返回，随后隐藏渲染环境把同一图形描述写成文件。

```cangjie role=trace
let window = PlotWindow(figure)
window.show()
FigureExport.renderToPng(figure, "latency-report.png", width: 800, height: 500)
println("交互检查结束，已导出 latency-report.png")
```

若两条路径需要不同尺寸，不必重建数据；尺寸属于渲染目标，图形描述仍然复用。纯批处理则直接调用 `FigureExport`，不要先打开窗口。

## 常见误解

- **“主题会自动让图可读。”** 主题只给默认视觉变量，标签、图表选择和信息密度仍由作者负责。
- **“导出就是窗口截图。”** 导出器可以在隐藏环境按指定尺寸重新渲染，不需要可见窗口。
- **“图例越全越好。”** 无意义或重复标签会占空间；图例应回答读者实际需要区分的身份。
- **“参考线应改变自动范围。”** 标注通常解释已有数据语境；需要显示远离数据的阈值时，应同时检查并显式设置范围。
