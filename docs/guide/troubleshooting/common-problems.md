# 排查常见绘图问题

## 先看现象

把问题先归入一类：项目无法编译；编译成功但窗口/导出启动失败；构造系列时直接报错且没有窗口；窗口出现却没有系列；对数轴上部分折线断开；全部非正数据直接报错；标注、图例或文字不可见；PNG 没有生成或物理尺寸与请求逻辑尺寸不同。不要同时修改数据、标度、主题和运行环境，那会让原因更难隔离。

## 可能原因

1. **构建边界**：`cjpm.toml` 路径没有指向 `plot` 根，或示例导入了未声明的直接依赖。
2. **运行环境**：SDL3/SDL3_ttf 动态库不在搜索路径。隐藏导出同样需要渲染运行时。
3. **组合遗漏**：系列已构造但没有 `axes.add`，面板未加入 `Figure`，或网格槽位与面板数量不匹配。
4. **数据域**：数组长度不一致会在系列构造时抛错；全部数据为空/无效会让面板缺少数据域；显式范围会裁掉数据；对数轴的部分非正值形成断点，而全部非正值会抛错。
5. **解释层**：标注在当前范围外、图例标签为空、文字与背景对比不足、窗口尺寸过小。
6. **文件路径**：目录不存在、权限不足、尺寸非法或循环覆盖同一文件。

## 诊断步骤

### 症状一：项目无法构建，或构建成功但窗口/导出无法启动

**如何诊断：** 先把构建与启动分开。运行 `cjpm build`；若这一步失败，只检查 `cjpm.toml` 是否指向 `plot` 项目根、`plot` 的兄弟 `sdl` 路径是否成立，以及导入和 API 签名，不要先处理 DLL。构建成功后再运行最小的[第一张折线图](../getting-started/first-chart.md)。若最小窗口也启动失败，或最小导出在创建隐藏窗口时失败，问题在 SDL 运行库、显示环境或原生渲染初始化，不在业务数据。

```powershell
cjpm build
cjpm run
```

**如何修复：** 路径错误时把依赖改到含 `cjpm.toml` 的项目根，并只导入公开包；签名错误按 API 参考修改调用。启动阶段缺少 SDL3/SDL3_ttf 时，把仓库提供的动态库复制到实际可执行文件目录，或加入当前进程的动态库搜索路径后重启终端。隐藏 PNG 导出也使用 SDL，不能因为没有可见窗口就省略运行库。

**如何确认：** `cjpm build` 应以状态码 0 结束。交互路径必须能打开最小窗口、显示固定折线并在关闭窗口后返回；导出路径必须能从调用返回并产生非空文件。只有构建成功、启动仍失败时才继续查动态库或显示环境，避免把两个阶段混成一个问题。

### 症状二：窗口出现但没有系列，或对数折线断开/直接报错

**如何诊断：** 先缩减为 `Figure.single`、一条固定系列和一次 `axes.add`，再打印 x/y 长度、有效值数量、最小值与最大值。构造 `LineSeries` 就报告 matching lengths 且窗口从未出现，说明两数组数量不同；窗口有坐标轴却没有系列时，检查系列是否真的传给 `axes.add`、数据是否为空/无效，以及显式范围是否把数据裁掉。标注或文字不见时，还要检查数据坐标是否落在当前范围内、图例标签是否为空及主题对比度。

对数轴要单独检查所有可见值大于零。进入绘图 API 前运行下面的最小探针，把“数据准备错误”和“渲染错误”分开：

```cangjie role=probe
let xs = [1.0, 2.0, 3.0, 4.0]
let ys = [12.0, 0.0, 18.0, -3.0]
println("x=${xs.size}, y=${ys.size}")
for (value in ys where value <= 0.0) {
    println("对数轴不能显示：${value}")
}
```

诊断输出按三种情况解释：

- **构造 `LineSeries` 就报 matching lengths，窗口从未出现**：打印两数组的 `size`，修正为相同数量后重新构造。这不是渲染故障。
- **对数折线只在个别位置断开**：数据中同时有正值和非正值；决定过滤、分面或改回线性轴，并在图注说明处理规则。
- **报告 “A log axis needs positive data” 且没有结果**：自动范围内所有值都不为正。不要把零替换为极小数，改线性轴或先修正业务数据。

**如何修复：** 先让 x/y 数量一致并确保系列、面板都已加入组合；为空数据给出“暂无数据”或跳过面板，不把回退域解释为观测。对部分非正值，明确选择过滤、分面或线性轴并记录规则；全部非正时改用线性轴或拒绝输入。阈值位于域外时显式扩大范围。零和负值若是业务中的真实观测，保留线性轴，不伪造一个可取对数的极小正数。下面片段使用与探针相同的数据并直接导出，应替代故障程序中的 `LogScale` 设置：

```cangjie role=fix
let xs = [1.0, 2.0, 3.0, 4.0]
let ys = [12.0, 0.0, 18.0, -3.0]
let (figure, axes) = Figure.single(title: "含零值的线性结果")
axes.yLabel = "原始值"
axes.add(LineSeries(xs, ys, label: "观测"))
FigureExport.renderToPng(figure, "linear-values.png", width: 600, height: 400)
```

在实际文件中，确保导入 `plot.{Figure, FigureExport}` 与 `plot.series.LineSeries`。

**如何确认：** 线性修复调用应成功返回，`linear-values.png` 非空并显示零和负值；若选择过滤后保留对数轴，重新运行探针时所有送入对数系列的值都必须为正，断点与图注中的过滤规则一致。重新逐项恢复第二系列、标注、主题和多面板，每恢复一步都观察结果；这能指出是哪项配置让问题复现。

### 症状三：PNG 没有生成、路径报错，或物理尺寸与请求值不同

**如何诊断：** 暂时去掉动态目录、批量命名与上传步骤，用[导出 PNG](../how-to/export-image.md)中的固定 `Figure`、当前目录文件名和正逻辑尺寸单独调用 `renderToPng`。尺寸非法属于 `PlotException`；SDL 初始化、隐藏窗口、渲染或捕获失败属于原生渲染路径；临时 BMP 读回或最终 PNG 不可写还可能传播文件系统异常。记录具体运行期类型与消息，不要只捕获 `PlotException` 后把路径或 DLL 失败漏掉。

若文件存在但物理尺寸与请求的逻辑尺寸不同，读取 PNG IHDR，而不是只看调用参数。`width`/`height` 是布局使用的逻辑尺寸；默认的 2 倍超采样目标可用时，600×400 通常落盘为 1200×800。目标不可用时渲染器会回退到直接绘制，可能得到 600×400；两种结果都不是编码器擅自改变纵横比。

**如何修复：** 在业务层先创建输出目录并检查权限，校验宽高为正，为每个数据集生成唯一文件名；补齐 SDL 动态库后再恢复隐藏导出。先用固定样例读取 IHDR，确认当前环境是 2 倍超采样还是 1 倍直接绘制，再按实测倍率反推请求的逻辑尺寸；批量任务对每个产物继续检查 IHDR。批处理边界可捕获 `Exception` 记录失败，但必须停止后续“上传成功”或“报告已生成”的流程，不能吞掉异常继续交付。

**如何确认：** 调用必须正常返回，目标文件存在且大于零，图片查看器能看到预期标题与系列。读取 IHDR 应得到预期物理宽高；固定路径成功后，再逐项恢复动态目录、批量命名、移动和上传，每一步都验证文件仍存在且未被循环覆盖。

## 修复方法

修复顺序固定为“构建边界 → 运行环境 → 最小组合 → 数据与标度 → 标注与主题 → 文件交付”。每个症状分支已经把对应动作与确认条件放在一起；若同时出现多个问题，按此顺序只改一个变量并回到该分支的“如何确认”，不要一次更换依赖、数据、主题和输出路径。

## 确认已经修复

三个分支的共同终点都是先让最小任务通过，再恢复原任务。窗口问题必须可打开、可关闭且系列可见；对数问题必须证明输入与所选标度一致；导出问题必须证明文件非空、尺寸正确且内容可读。只看到“命令没有报错”不足以证明视觉结果正确，也不能代替各分支已经列出的具体验收项。

## 避免再次发生

保留一个固定数据的冒烟示例；把数据清洗与图形构建分开；为对数和显式范围写输入断言；批量导出使用确定性文件名并检查结果；升级主题或系列组合后重新生成视觉基线。验证报告应明确哪些 GUI 示例只编译、哪些实际运行或导出。
