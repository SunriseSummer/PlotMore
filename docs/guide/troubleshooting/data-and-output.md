# 数据、坐标与导出排错

## 先看现象

本页处理程序已经能编译运行、但结果不正确的情况：面板为空或系列缺失，数据被压扁、裁切或对数轴报错，以及导出调用返回后文件不存在或内容异常。先确认是数据层、坐标层还是文件层，再做最小修复。

## 可能原因

空图常因数组长度不一致、忘记 `axes.add`、非有限值或系列不可见；范围异常常因错误限制、对数域含零负数、类别标度放错轴；输出异常常因工作目录误判、父目录不存在、权限或扩展名问题。FigureExport 只负责渲染已有 Figure，不会替应用修复数据语义。

## 诊断步骤

### 症状一：窗口或 PNG 有坐标框，但没有预期系列

先检查 Figure 面板数、每个 Axes 是否执行 `add`，再打印 x/y 数组长度和几个边界值。不要立刻固定巨大范围，因为那可能把真正的长度或非有限值问题隐藏起来。下面探针让长度与首尾值在构造系列前可见；确认输出中的长度相同，且数据确实落在预期数量级。

```cangjie role=probe
let xs = [1.0, 2.0, 3.0, 4.0]
let ys = [12.0, 15.0, 14.0, 18.0]
println("x=${xs.size}, y=${ys.size}")
println("first=(${xs[0]}, ${ys[0]}), last=(${xs[3]}, ${ys[3]})")
```

数据正确后，使用下面修复片段明确创建、加入并核对系列。关键不是重新实例化窗口，而是让系列进入正确面板。运行后应确认 `allSeries().size` 为 1，随后窗口或导出图片中出现四个点组成的折线。

```cangjie role=fix
let (figure, axes) = Figure.single(title: "数据检查")
let series = LineSeries(
    [1.0, 2.0, 3.0, 4.0],
    [12.0, 15.0, 14.0, 18.0],
    label: "有效数据"
)
axes.add(series)
println("series=${axes.allSeries().size}")
```

### 症状二：系列可见，但被压成一条线、被裁切或对数轴失败

检查原始最小最大值，再检查 `setXLimits`、`setYLimits`、当前 Scale 域和 `hasView()`。若使用 LogScale，打印或扫描所有值，确认域和要显示的数都大于零。临时调用 `resetView`，比较复位前后结果；再移除显式限制，观察自动范围是否恢复正常。修复后确认主要数据落在面板内部、刻度单位合理、多个待比较面板使用同一范围。若异常值真实存在，不要静默丢弃，应在单独面板、标注或文字中说明。

### 症状三：终端显示导出完成，但目标文件不存在、为空或尺寸错误

先打印进程当前工作目录和最终路径，检查父目录确实存在且可写，文件名使用 `.png`。在 `renderToPng` 返回后再检查文件，不要并发读取尚未完成的输出。打开文件并读取实际像素尺寸；参数是逻辑宽高，渲染器可能使用超采样，交付约束应以 PNG 文件头为准。修复后确认文件存在、大小非零、图像查看器能打开、标题与四周没有裁切，并记录最终绝对路径。若连简单图也导不出，返回[构建与运行排错](build-and-runtime.md)检查隐藏 SDL 环境。

## 修复方法

数据层修复应保持数组一一对应、过滤或替换非有限值，并显式把系列加入目标面板。坐标层先恢复自动范围，再只添加确有业务依据的 Bounds 或 Scale。文件层使用已存在的输出目录、明确扩展名和同步调用。每次只改一层并重新运行，保留修复前后的可观察差异。

## 确认已经修复

对 GUI，确认系列、图例、范围和交互复位都符合数据；对 PNG，确认程序退出码为 0、文件存在且可打开、像素尺寸与交付要求一致。对场数据还要确认 ColorBar 与系列值域一致；对分布比较要确认各组使用共同范围或清楚分面。将这些结果写入验证记录，不要仅写“看起来正常”。

## 避免再次发生

在构图前验证数组长度、有限性和业务范围；为可复用构建器保留固定小数据测试；统一管理导出目录和文件命名。多面板比较显式共享 Bounds，连续色图共享值域和 ColorBar。批处理在上传前检查文件存在和大小，GUI 测试则把“可编译但不自动运行”与真正的运行验证区分记录。
